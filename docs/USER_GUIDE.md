# User Guide

## Install for development

```bash
./scripts/bootstrap_client.sh
cd apps/mobile
flutter run
```

For desktop development, run the same Flutter package on a desktop target:

```bash
cd apps/mobile
flutter run -d linux
flutter run -d macos
flutter run -d windows
```

For server development:

```bash
cd services/server
dart pub get
dart run bin/server.dart
```

## First-run setup

AetherTune follows the device language by default for the application title, main navigation, and first-run setup. In Options, choose **System default**, **English**, **Turkish**, or **Arabic** to keep an explicit language override across app restarts and portable backup restore. English, Turkish, and Arabic are available today; Arabic automatically uses right-to-left layout. Most feature screens remain English while the localization catalog is expanded.

On first launch, choose **Open Library** to begin importing local audio, **Open Sources** to set up supported legal catalogs and user-owned servers, or **Start at Home** to explore first. Setup never enables a network provider by itself and can be skipped. Use **Run setup again** in Options whenever you want to revisit these choices.

## Import local music

1. Open the app.
2. Tap the library-add icon in the top-right corner to select one or more audio files, or tap the folder icon to scan a folder recursively.
3. AetherTune imports supported audio files, reads basic ID3v1 MP3 title/artist/album tags, ID3v2 MP3 title/artist/album/genre text tags plus APIC/PIC artwork, FLAC Vorbis comment title/artist/album/genre tags plus picture artwork, M4A title/artist/album/genre metadata atoms plus `covr` artwork, and WAV RIFF INFO title/artist/album/genre tags during folder scans, attaches matching basename `.lrc` or `.txt` lyric sidecars without replacing lyrics you already saved, parses common filename metadata such as leading track numbers and `Artist - Title` when tags are missing, and groups folder-scanned tracks by their parent folders.
4. Tap a track to play it.

## Home

The Home tab shows local feed sections for recommendations, mood/activity mixes, in-progress tracks, recently played tracks, newest local additions from followed artists, radio seeds with local matches, most played tracks, favorites, saved subscribed podcast episodes, recently added tracks, and local charts. When a configured Jellyfin or Navidrome/Subsonic account is available, **From your servers** can explicitly refresh bounded server shelves. Jellyfin supplies recently added music albums; Navidrome/Subsonic supplies newest, frequently played, recently played, and random albums; both retain a playlist shelf. In a server's **Artists** tab, use the heart button to follow or unfollow an artist. When that server's recently-added shelf contains a followed artist, Home adds a separate **from artists you follow** shelf without another request. On a Navidrome/Subsonic shelf that has another server-defined page, choose **Load more** to append distinct albums; failed continuation keeps the current shelf and lets you retry. These are server-ordered discovery lists, not AetherTune personalized recommendations. Other catalog adapters fall back to generic albums and playlists. AetherTune does not contact those servers merely because Home opened, disables the refresh in offline mode, and opens a selected result through the existing server collection page. Available results remain visible if another server section fails, without displaying credential-bearing provider errors. The "For you" section uses favorites and recent playback as local taste signals while skipping just-played tracks, and each recommendation states which favorite/recent metadata, favorite status, unplayed status, or recent addition caused it to appear. In Options, **Use favorites in For you** and **Use listening history in For you** independently control those signals; when both are off, the section uses recent additions without reading either taste source. Focus, Energy, Chill, Workout, and Sleep mixes are generated from playable library metadata. Tap a mix header to open its full live generated list, play the mix, or save its current order as an editable playlist. The chart range selector can show all-time, last-7-days, last-30-days, or last-year top tracks, artists, albums, and genres from playback history. Tap a track to play that section as the queue, or open its menu for lyrics, playlists, metadata edits, favorites, removal, radio, and **Copy share text**.

On desktop-width windows, AetherTune uses a labeled left navigation rail instead of bottom navigation so Home, Library, Playlists, History, Sources, and Options stay visible beside the content and player bar. Use `Ctrl+1` through `Ctrl+6` to select those tabs directly, or `Alt+Left` and `Alt+Right` to cycle through them. Wide desktop windows also show a queue pane; drag its left edge to resize it, then select, remove, or reorder queued tracks. The width is remembered on that device. In Options, choose **Desktop density**: **Comfortable** preserves the standard control spacing, while **Compact** fits more rows on Linux, macOS, and Windows without changing phone or tablet spacing. The choice is stored on that device. Turn on **Minimize to tray on close** to keep playback running after the window closes; use the tray menu's **Quit AetherTune** action to exit.

## Search

Use the search box on the Library tab. Search matches:

- title
- artist
- album/folder
- genre
- source
- saved plain or LRC lyrics

Saved synced LRC lyrics are searched by their visible lyric text, not by timestamp tags. Library, playlist, suggestion, browse, and saved-lyrics searches tolerate small typos in meaningful search terms. Suggestion chips below the search bar come from submitted searches, recently played tracks, and library metadata. Use the sort button in the search bar to order the library by recently added, title, artist, or album. Use the cloud-off button to show only local files that can play while offline mode is enabled. Use the Artists, Albums, Genres, Sources, and Folders chips below the search bar to browse grouped library views. Artist and album groups open full pages with representative artwork, live collection stats, Play, Shuffle, Radio, Save playlist, text/visual sharing, and normal track actions. **Radio** starts a local collection-aware queue anchored by favorite and play-history signals, then fills it with the remaining collection and related artist/album/genre matches; use the confirmation bar's **Save playlist** action to keep that generated queue as an editable playlist. Artist pages also show local albums, follow/unfollow controls, and related artists; album pages can open their single local artist and show related albums. Related collection rows state whether artist, album, or genre metadata matched. Follows are included in backups and library sync snapshots; following an artist from a self-hosted catalog can filter that server's explicit recently-added Home shelf, but does not create a provider-native remote creator/channel subscription. The Folders chip shows a recursive tree with parent folders, child folders, aggregate track counts, and descendant playback queues. Importing an audio folder also adds it to local monitoring: where the platform filesystem supports it, AetherTune debounces audio and `.lrc`/`.txt` changes, rescans it while the app runs, and refreshes it when the app opens. In Options, refresh a watched folder manually or stop monitoring it without removing its imported tracks. Open a browse group or folder node and use the share icon to copy privacy-safe share text; local file paths are not included. Open a track menu and choose **Similar tracks** to find playable local tracks with matching artist, album, or genre metadata; the sheet explains each match and can play the results as a queue.

## Metadata

Open a track menu and choose **Edit metadata** to update the saved title, artist, album, or genre. These edits update library search, browse groups, playlists, suggestions, and backups. For a local-library track, choose **Artwork** to select a validated PNG, JPEG, GIF, or WebP image, set an http/https cover URL, or restore scanned embedded artwork. M4A tracks also offer **Write cover to M4A file**: choose a PNG or JPEG under 512 KiB, confirm the file update, and AetherTune replaces the embedded cover and refreshes its scanned artwork immediately. Picked private images are stored only on the current device; backups, library sync, and playlist documents use embedded artwork or safe web URLs instead of private file paths. For local MP3, FLAC, M4A, and WAV files, AetherTune separately asks whether to write the values to the file. MP3 writing uses ID3v2.3/v2.4 text frames so Unicode metadata and custom genres survive, keeps an ID3v1 compatibility tag for older players, preserves supported artwork and other non-editable ID3v2 frames, and aborts before changing files with unsupported tag layouts. FLAC writing updates standard Vorbis comments while preserving non-comment metadata blocks, including artwork, custom comments, and audio bytes; malformed comments are left unchanged. M4A writing updates standard title/artist/album/genre metadata atoms and can replace the embedded PNG/JPEG cover, preserves other metadata items and media bytes, and uses an atomic replacement. When a standard front-loaded `moov` changes size, AetherTune repairs validated `stco`/`co64` chunk offsets; malformed or fragmented atom layouts are refused before any mutation. WAV writing updates RIFF INFO fields while preserving unrelated RIFF chunks, custom INFO fields, and audio bytes; malformed INFO lists are left unchanged, and legacy INFO text uses `?` for characters outside Latin-1.

Ogg Vorbis and Opus metadata writes are also available for standard isolated comment-header pages. AetherTune preserves custom comments and media pages, recalculates the edited Ogg page CRC, and refuses shared, continued, malformed, or nonstandard comment layouts before changing a file.

## Duplicate resolver

Open the Options tab and choose **Resolve duplicates** when duplicate groups are found. AetherTune can match duplicate library entries by same local path, scanner content hash, provider item, stream URL, or metadata plus known duration. Choose **Keep** on the track you want to preserve; playlists, favorites, lyrics, history, and playback progress are merged into that kept track.

## Favorites

Open a track menu and choose **Favorite**. Use the heart button in the search bar to filter favorites.

## Queue controls

When you play a track from the library, the current filtered list becomes one lazy native playlist so the audio engine can prepare and transition to the next item without AetherTune reloading each track. Android, iOS, and macOS use the native `just_audio` backends; Windows and Linux use the bundled MediaKit audio backend with playlist prefetch. Open a track menu and choose **Start radio** to build a local seed queue from playable tracks with matching artist, genre, or album, ranked by favorites and play history. The start confirmation offers **Save playlist** to retain that generated queue as an editable playlist. Use **Copy share text** from a track menu to copy track metadata and a web link when one exists; local paths are redacted. The copy confirmation also offers **Share** to open the native platform share sheet with that same privacy-safe text. The **Similar tracks** sheet can also play its matched result list as the queue. AetherTune restores the current queue when you reopen the app. Tap the artwork/title area in the compact player to open the full Now Playing screen. It shows large artwork, elapsed and remaining time, seeking, queue position, favorite, lyrics, queue, shuffle, repeat, volume, and transport controls; swipe artwork left for the next track or right for the previous track. Saved current tracks can also store chapter lines as a timestamp and title; the **Chapters** panel highlights the active section and seeks directly to any marker. The compact player keeps only the essential controls on narrow phones and exposes the wider action set on larger screens. Use the queue to move tracks up/down or remove upcoming tracks while preserving the current position, or use the compact player's playlist-add button on larger screens to save the current queue as a playlist. Offline mode rebuilds the native queue with local files only.

Podcast RSS episodes with inline Podlove Simple Chapters or same-origin Podcasting 2.0 chapter documents show their markers automatically in Now Playing. The seek timeline displays proportional chapter ticks beneath its slider. AetherTune does not request chapter documents hosted outside the feed's declared origin.

On Android, open **Options > Audio effects** to enable the native equalizer and choose Flat, Bass boost, Vocal, or Treble. Once a track has loaded, the dialog shows the center frequencies and gain range reported by that device; moving any band creates a persisted Custom profile. Equalizer state and curves apply to both normal playback and the incoming crossfade player. **Volume boost** separately controls the Android loudness enhancer from 0 to 12 dB. **Spatial audio** controls Android's native virtualizer from 0% to 100% and follows both normal playback and an incoming crossfade session. It is intended for compatible headphones and speakers, and is independent from ReplayGain-based loudness normalization. These controls are hidden when the active playback backend does not support them.

On Android, the media notification and headset buttons show the current title, artist, album, artwork, queue position, playback position, and available previous/play-pause/next/stop actions. Android Auto can browse the current queue and **Library > All tracks**. Selecting a queued item uses the current queue; selecting a library track uses the same resolver, offline policy, queue persistence, output settings, and playback controls as the app. On iOS and macOS, the same queue metadata and controls are published to Control Center and Now Playing surfaces. Jellyfin and Navidrome/Subsonic artwork is copied from validated provider bytes into bounded private temporary files before system publication, so authenticated artwork URLs and credentials never enter media metadata. These runtime files are reused, pruned, omitted from queue/library JSON, and removed with the account. System seek, repeat, and shuffle commands are routed to the same queue engine as the in-app controls. Android and iOS are configured for background music playback; operating-system battery policy, audio interruptions, route changes, and device-specific lifecycle behavior still need physical-device validation before release.

## History

The History tab shows a local listening recap with library totals, favorites, play count, estimated listening time, responsive top-track and top-artist charts, monthly and yearly calendar recap cards, top tracks, top artists, top albums, top genres, recently played library tracks, individual play history entries, and last played times. Use the bookmarks icon to save the current date range and history search as a named view; saved views can be applied, updated to the current filters, renamed, or deleted and are included in backups. Use the image icon on a monthly or yearly card to preview a share visual, choose the Midnight, Daylight, Signal, or Monochrome color swatch, and save that exact theme as a PNG through the platform file dialog. The selected recap theme remains active after restart and travels in backups and manual sync snapshots. Use **Pause listening history** to stop saving new plays and resume progress while keeping existing history until you clear it. Use the range selector to view all-time, last 7 days, last 30 days, or last-year stats; use the export button to copy the selected range as JSON or CSV. Use **Search listening history** to filter recently played tracks and individual play entries by title, artist, album, genre, source, folder, or saved lyrics. Tap a track to play it again, use the delete icon beside a play history entry to remove only that play, or use the clear button to remove all local listening history.

The Listening calendar in History maps each day in the selected range by play count and estimated listening time. All-time history displays the most recent 84 days; longer selected ranges can be scrolled by week.

## Playlists

Use the Playlists tab to open built-in smart playlists for favorites, recently added tracks, recently played tracks, and most played tracks. Create custom smart playlists with search text, exact source ID, artist, album, and genre, optional minimum/maximum duration in seconds, favorites-only, minimum play count, minimum days since last play, sort mode, and result limit rules; choose **Match all** when every active criterion must pass or **Match any** when one active criterion is enough. Add nested rule groups to combine those modes recursively, up to 25 groups per level, 50 rules per group, and eight group levels. AetherTune updates their tracks dynamically as the library changes and preserves that matching choice and rule tree in device storage, backups, and manual sync. The days-since-last-play rule also matches tracks that have never been played. Search text matches saved lyrics as well as metadata. You can also create, rename, choose private local artwork, delete, open, search within, reorder, import, export, and play manual playlists. Use a manual playlist menu's **Move to folder** command to create a folder, move the playlist, or clear the field to return it to **Unfiled**; the folder chips filter the manual playlist list and the setting travels in JSON exports, backups, and manual sync. Add tracks from the Library tab by opening a track menu and choosing **Add to playlist**. Use the import button to choose a UTF-8 JSON, M3U, or CSV file or paste its content; imports link tracks that already exist in your local library and preserve AetherTune playlist artwork URLs from JSON exports. Use a playlist menu to export JSON/M3U/CSV, then save a portable file through the platform dialog or view the document, choose **Copy share text**, or open **Artwork** to choose a PNG, JPEG, GIF, or WebP image from your device, set an http/https image URL, or remove artwork. Picked images are copied into private app storage and are not included in backups, playlist files, or cross-device sync; web artwork URLs remain portable. Open a playlist and use the search box to find tracks by title, artist, album, or saved lyrics; use a track menu to move a track up, move it down, or remove it.

**Copy import link** creates a bounded `aethertune://playlist` or `aethertune://smart-playlist` URL. Opening one from Android, iOS, macOS, Linux, or Windows waits for the local library to load, imports the safe document, opens Playlists, and shows the imported name or an existing validation error. Links never contain private paths, stream URLs, credentials, or picked local artwork.

When a manual playlist has no selected image, AetherTune generates its artwork from up to four playlist tracks in their playlist order. A selected local or web image always overrides the generated collage.

For selected manual playlist artwork, choose **Crop and position** from the playlist artwork menu to pan within a square preview, set a 1x-3x zoom level, reset, and save a non-destructive crop profile. The profile persists with the playlist and resets when the selected image changes or is removed; it is safe to include in portable playlist data even when the private image file itself stays on the current device.

## Lyrics

Open a track menu in the Library tab and choose **Lyrics**. You can save plain text lyrics, import a UTF-8 `.txt` or `.lrc` file, paste LRC timestamped lyrics, or choose **Search online** to search LRCLIB. The search sheet discloses the contacted domain and metadata sent, ranks plain and synced matches against the track metadata and duration, and lets you select one result; instrumental records without lyrics cannot be selected. The selected lyrics are stored locally with LRCLIB provider/record attribution, shown in the editor and now-playing lyrics header, included in backups, and added to copied share excerpts. Typing into or importing over provider lyrics changes the source back to manual. Offline mode searches only persisted LRCLIB results for the same query and never contacts the provider. In **Options**, choose a 1, 7, 30, 90, or 180-day retention window for cached LRCLIB searches, or clear them at any time. Folder import attaches matching `.lrc` or `.txt` sidecars beside local audio, preferring LRC. It can otherwise import bounded embedded MP3 ID3v2 `USLT`/legacy `ULT`, FLAC/Ogg/Opus `LYRICS`/`UNSYNCEDLYRICS`, or M4A `©lyr` lyrics. Saved lyrics and sidecars always take precedence. Use the save icon in the editor to write the full UTF-8 lyrics document through the platform file dialog; it suggests `.txt` for plain lyrics and `.lrc` for synchronized lyrics. The image icon previews a bounded, attributed lyric card and saves it as a PNG. You can also copy the full lyrics export text, edit or delete lyrics later, and copy a bounded lyrics share excerpt. **Share selected lines** opens a picker for one to eight visible lines; timestamp labels are stripped from synchronized LRC excerpts. The same selected-line action is available from Now Playing. Saved lyrics are included in Library, playlist, and custom smart playlist searches. While a track is playing, use the lyrics button in the player bar to open now-playing lyrics; LRC lines highlight and scroll with playback.

## Backup and restore

Open the Options tab and choose **Export backup**, then choose **Save backup file** to write a versioned UTF-8 JSON file through the platform file dialog. **View backup JSON** remains available when you need to inspect or copy the document. Choose **Restore backup**, then choose **Choose backup file** or **Paste backup JSON** to replace the local library, locally followed artists, playlists, lyrics, listening history, saved history views, submitted search history, pause-listening-history and recommendation-signal preferences, theme, accent, and language preferences, offline mode, queued offline media requests, podcast feed subscriptions with cached episode catalogs, podcast refresh state, and podcast episode progress with that backup. Backups preserve queue metadata and local cache paths, not copied media bytes.

## Sleep timer

Tap the moon icon in the top-right corner and choose a preset duration, choose **Custom duration** and enter 1 to 1440 minutes, or choose **Stop at end of current track**. Turn on **Fade out before stopping** and choose a 10-second, 30-second, 1-minute, or 2-minute fade duration for timed sleep timers. Playback stops automatically when the timer or selected track ends.

## Cross-device library sync

In Options, configure a self-hosted AetherTune server with its URL, a device name, and an operator-provided bearer token. Current servers can issue a separate revocable token for each phone or desktop under one managed account; use the same account for devices that should share a library, but never copy one device token to another. The app tests the connection and authenticated identity before retaining the token in secure storage. Options then shows the account display name/ID and managed device name, without showing the token or its server-side ID. Use the pencil action advertised by current managed servers to rename the shared account and this device; the device name also becomes this installation's upload attribution. Use refresh after another device or operator updates account metadata. Older servers without the identity endpoint remain compatible and show identity as unavailable, while servers that identify a managed account but do not advertise editing omit the pencil action. Use the upload and download controls to move a portable library snapshot between devices. The snapshot includes library metadata, manual playlists, lyrics, listening history, and valid playback progress. Downloading asks for confirmation because it replaces the portable library state; matching local files and device-specific cache settings stay on the device. The delete server snapshot action asks for confirmation, leaves this device unchanged, records the reset as a new server revision, and disables automatic uploads so the cleared copy is not recreated. Turn on **Automatic foreground uploads** to upload at most every 15 minutes while AetherTune is open. It never downloads, overwrites, or merges automatically. Turn on **Sync active queue** only when you want to transfer the queue between these devices: it sends ordered IDs for tracks already in the synced library, the current item, and its latest change time. Each device reuses its own files and provider resolution, never receives media URLs, local paths, credentials, or playback position, and stays stopped after receiving a queue. When another device has uploaded a newer revision, choose the server copy, explicitly replace it with this device, or choose **Merge both**. Merge keeps independent records from both snapshots, combines matching playlist memberships, uses newer progress and lyric timestamps, keeps this device for same-ID metadata conflicts and scalar preferences, and selects the newest opted-in queue. Sync is paused in Offline mode and does not transfer provider credentials, runtime stream URLs, watched folder paths, or cached media.

## Options

The Options tab contains playback settings such as shuffle, repeat mode, a persisted default playback speed from 0.5x to 3x, a persisted independent playback pitch from 0.5x to 2x on Android, Linux, and Windows, persisted 5/10/15/30/45/60-second backward and forward skip intervals, and a persisted 0-100% playback volume, plus a **Language** selector for System default, English, Turkish, or Arabic, a **Theme** selector for System, Light, Dark, or AMOLED, and an **Accent color** selector with persisted swatches. On Android 8 or later, choose **Pin playback shortcut** and select Previous track, Play or pause, or Next track to ask the launcher to pin that transport command. The full player exposes the same default speed, pitch, and volume controls, plus track-speed and track-pitch controls that can apply a device-local override to the current track or return it to the default without changing that default. The applicable pitch override follows the current track through queue transitions. Dedicated rewind/forward controls use the configured intervals. Unsupported hosts omit the pitch controls. **Use favorites in For you** and **Use listening history in For you** independently control local recommendation inputs without deleting favorites or history. Turn on **Pause listening history** to stop new plays and resume progress from being stored until the toggle is turned off. Turn on **Offline mode** to pause network-backed source searches, feed refreshes, and saved stream playback from every player surface while keeping local file playback available. The Offline queue section shows provider-approved cache/download requests, lets you pause or resume queued requests, lets you turn on **Automatic foreground downloads** to process a bounded batch of eligible items sequentially while AetherTune is open, lets you cache queued direct media URLs into private app storage with post-write checksum verification, resumes retried direct HTTP(S) cache writes from saved `.part` bytes when the server supports Range requests, export verified cached media to a user-chosen folder, shows private cache usage, lets you set the private cache limit from 50 MB to 50 GB, lets you set per-provider private cache quotas for queued providers, and lets you trim the private cache to the app limit, clear cached media, remove one item, or clear the queue. After successful cache writes, AetherTune automatically evicts the oldest private cached files when usage exceeds a provider quota or the app limit. **Local diagnostics** shows the bounded, redacted error reports saved on this device; use its export action only when you choose to share a support file, or clear the reports at any time. The app never uploads those reports. AetherTune restores those playback settings, recommendation-signal and pause-listening-history preferences, theme, accent, and language preferences, offline mode, offline cache limits, cache metadata, paused queue state, and the offline queue when you reopen the app.

## Provider plugins

The Sources tab explains the provider model and shows capability/privacy disclosure for enabled adapters. When offline mode is on, network-backed source searches, feed refreshes, and stream playback actions are paused; saved stream-only tracks are also blocked by the player until offline mode is turned off. Use the offline-media menu beside source results to queue a cache or download request; eligibility is gated by each provider's declared capabilities and privacy disclosure. The Options tab can pause/resume approved queue entries, including stopping an active foreground transfer while retaining private `.part` bytes for a later HTTP Range resume, cache direct media URLs into private app storage with stored byte-count/checksum metadata, retry interrupted direct HTTP(S) cache writes from saved `.part` files, export verified cached files to a user-chosen folder, set app/provider private cache limits, and trim or clear those private cache files later. Background download jobs are still roadmap work. The current app includes:

- Custom JSON catalogs: add a declarative catalog URL, its optional media/artwork hosts, and a local description in Sources. HTTPS is required by default; a trusted LAN HTTP endpoint requires explicit consent. AetherTune downloads at most 2 MiB/500 tracks, filters searches on-device, does not follow catalog redirects, and rejects credential-bearing or off-list media URLs. These catalogs are data-only and cannot download or execute provider code.

- Provider search: search the local library and enabled legal adapters together with typo-tolerant result scoring, then play playable results, save any result to the local library, or queue provider-approved cache/download requests. With a configured Jellyfin or Navidrome/Subsonic account, entering at least two characters also shows bounded artist, album, and track suggestions after a short pause; select one to run the normal unified search for that value. Choose **Load more provider results** to append the next available page from each provider. Results are deduplicated and re-ranked across pages. If one provider fails, successful providers still advance, all prior rows stay visible, and **Retry failed providers** reuses only the failed cursors. Starting a new search discards stale replies from the old query. In offline mode, provider search pages local-library results only, sends no type-ahead request, and avoids network providers; switching offline during an online result set disables its continuation.
- Local Files: working.
- Demo Provider: template with declared search capability and no network access.
- Podcast RSS: in Sources, search the public Apple podcast directory by name or topic, then select **Subscribe** to hand its RSS URL to the same validated feed flow. The search field clearly discloses that its query is sent to `itunes.apple.com`, returns only valid HTTP(S) RSS URLs, and does nothing while offline mode is on. You can also add a legal RSS feed URL directly, import/export OPML, then refresh, play, resume, save, queue/cache checksum-verified eligible feed enclosures with direct HTTP(S) retry resume, trim/clear private cached media, set provider cache quotas, or remove feed episodes. Each refresh retains up to 100 episode records per feed for offline browsing and the Home subscribed-episodes feed; backup and same-feed sync merge preserve them. While AetherTune is open, due feeds refresh on launch, resume, and a 15-minute cadence; the worker skips fresh feeds, isolates failures, and makes no network request in offline mode. OS-background refresh and download jobs remain roadmap work.
- Radio Browser: AetherTune discovers a public Radio Browser API mirror with fallback to the bundled default, searches public internet radio stations in Sources, filters by country/language/tag/codec/bitrate, and uses **Load more stations** for offset-based continuation. Transient directory/search failures retry twice with short backoff before the visible error action appears; invalid directory data fails immediately. A failed continuation keeps earlier stations and offers a retry. Open each station's detail page for directory status, tags, codec/bitrate, homepage, stream URL, play, save, and validation actions. Station plays are reported once to Radio Browser's click endpoint. Live radio streams do not declare cache/download eligibility.
- Internet Archive: search public archive audio in Sources, filter by collection/subject/creator/year, apply returned facet chips, and use **Load more archive results** to continue through a result set. A failed continuation keeps the earlier results and offers a retry. Open a result to inspect its collection, subjects, license, and playable files. From that detail view, play or save one/all files, or queue/cache checksum-verified public files with direct HTTP(S) retry resume; private cached media can be quota-limited, trimmed, or cleared in Options. Background download jobs remain roadmap work.
- Jellyfin: choose **Add Jellyfin**, enter a display name, complete server URL, Jellyfin user ID, and API key, then choose **Test and save**. A successful account joins provider search. Select its account card, or choose **Browse library** from its menu, to open filterable **Artists**, **Albums**, and **Playlists** tabs. Each tab loads a bounded first page; choose **Load more** to append the next page. A failed continuation leaves every prior row visible and offers **Retry** at the same position. Pull to refresh restarts that tab from its first page. Artists open their albums; albums and playlists open tracks. Artist and album pages expose **Start radio**, and tracks expose it from their menu; Jellyfin supplies a playable Instant Mix for that seed, while AetherTune keeps a selected track first and resolves the queue only at playback. Authenticated artwork appears in those views, saved library rows, the mini-player, Now Playing, and system media surfaces without placing the API key in an image URL. In **Playlists**, use **Create playlist** or a playlist's menu to rename/delete it. From an album track's menu choose **Add to remote playlist**; inside a playlist, use track menus to move tracks up/down or remove them after confirmation. Use **Play all**, **Save all**, a track row, or its menu to play, save, queue an offline cache, or queue a download. Use **Edit and test** for non-secret account metadata, **Rotate credential** to confirm and test a replacement API key before the vault changes, or **Remove** to delete the account. Manual AetherTune snapshots can carry local saved metadata and playlists, but intentionally exclude the Jellyfin account, API key, and remote provider playlist state.
- Navidrome/Subsonic: choose **Add Navidrome**, enter a display name, complete server URL, username, and password, then choose **Test and save**. AetherTune verifies the account through `ping.view`, includes it in provider search, supplies bounded artist/album/track type-ahead through documented `search3.view` results, and exposes the same filterable **Artists**, **Albums**, and **Playlists** browser from the account card or **Browse library** menu. The Albums tab uses **Load more** with retained-results retry; Artists and Playlists use the protocol's complete-list endpoints, which do not define offsets. Requests derive a fresh salted token from the vault password instead of transmitting the reversible encoded password. Artist and album pages expose **Start radio**, and tracks expose it from their menu. Track/album seeds use `getSimilarSongs.view`; artist seeds use `getSimilarSongs2.view`. Navidrome requires its Last.fm integration for those endpoints. Artist, album, playlist, play/save, artwork, offline-queue, and remote playlist create/rename/delete/add/remove/reorder actions use the documented Subsonic endpoints; selected songs resolve through `stream.view` only when playback or caching needs a stream, and validated artwork reaches system media surfaces through a private local file. Use **Rotate credential** to confirm and test a new password before replacing the vault value. Manual AetherTune snapshots can carry local saved metadata and playlists, but intentionally exclude the Navidrome/Subsonic account, password, and remote provider playlist state.

Self-hosted accounts require HTTPS by default. If a LAN-only server uses plain HTTP, AetherTune shows a specific warning and requires explicit consent because credentials would travel without TLS. Account metadata and safe artwork IDs/tags are stored separately from secrets; API keys/passwords use the operating system credential vault and are not included in AetherTune JSON backups. The metadata editor never displays the saved secret. Rotation requires the replacement twice, tests it against the server before writing, and restores the old vault value if the secure write fails. A successful rotation invalidates bounded in-memory and private-file artwork, strips old runtime media URIs from the queue, and re-resolves active provider tracks while preserving playback position and play state when possible; a track that cannot be resolved remains stopped and metadata-only. Authenticated image URLs never enter the track model or Flutter image cache. Authenticated stream URLs exist only while preparing playback or an approved offline request, are stripped from persisted queues, and are rebuilt from the vault after restart. Offline mode disables account browsing without issuing catalog requests. Removing an account deletes its vault entry and queued tracks. If the credential vault is unavailable, account actions remain disabled and Sources shows the storage error.
